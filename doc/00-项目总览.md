# 基于 Streamlit 与迁移学习的食物图像识别与热量估算系统 - 项目总览

## 1. 项目名称

**基于 Streamlit 与迁移学习的食物图像识别与热量估算系统**

本项目面向日常饮食健康管理场景，设计并实现一个兼容 PC 浏览器和移动端浏览器的交互式 Web App。用户上传或拍摄食物图片后，系统使用 PyTorch 深度学习模型识别食物类别，再结合用户输入的食物重量和本地热量数据库估算总热量，并调用 GPT-5 API 生成个性化饮食建议。

## 2. 项目背景

在日常生活中，用户经常需要记录饮食热量，但人工查询食物热量表效率较低，且很多用户难以快速判断当前食物的大致热量。本系统通过图像识别技术降低记录门槛，使用户只需上传食物图片并输入大致重量，即可获得类别识别、热量估算和饮食建议。

需要明确的是：本系统不直接从图片中精确估计食物重量，也不提供医学或营养诊断。热量结果属于基于食物类别和用户输入重量的辅助估算值。

## 3. 项目目标

- 构建一个基于 Streamlit 的交互式应用，兼顾 PC 端和移动端浏览器访问体验。
- 使用 Food-101 子集训练 ResNet18 迁移学习模型，实现常见食物图像分类。
- 根据食物类别、每 100g 热量和用户输入重量估算总热量。
- 接入 GPT-5 API，生成简短、实用、非诊断性质的饮食建议。
- 使用 SQLite 保存食物热量表、识别历史和 GPT 建议记录。
- 输出可支撑课程设计报告、演示视频和源码开发的完整方案。

## 4. 核心技术栈

| 类型     | 技术                  |
| ------ | ------------------- |
| 开发语言   | Python              |
| 交互界面   | Streamlit           |
| 深度学习框架 | PyTorch             |
| 图像识别模型 | ResNet18 迁移学习       |
| 数据集    | Food-101 子集         |
| 图像处理   | Pillow / OpenCV     |
| 数据处理   | Pandas / NumPy      |
| 数据存储   | SQLite              |
| 大模型服务  | OpenAI GPT-5 API    |
| 开发环境   | PyCharm + Miniconda |

## 5. 主要功能

| 功能       | 说明                                    |
| -------- | ------------------------------------- |
| 图片上传与预览  | 支持 PC 文件上传和移动端相册/拍照上传，支持 jpg、jpeg、png |
| 食物类别识别   | 使用 ResNet18 模型输出预测类别、置信度和 Top-3 结果    |
| 热量估算     | 根据食物热量表和用户输入重量计算估算热量                  |
| GPT 饮食建议 | 根据食物名称、重量、热量和用户目标生成建议                 |
| 历史记录     | 保存识别时间、图片名、类别、置信度、重量、热量、建议            |
| 食物热量表    | 展示系统支持类别的中文名、英文名、每 100g 热量            |
| 统计分析     | 展示历史总热量、按日期汇总热量和摄入趋势                  |
| 系统说明     | 介绍项目、使用方法、估算限制和注意事项；系统说明页面不展示技术栈      |

## 7. 创新点

- 将食物图像识别与热量估算结合，形成完整生活应用场景。
- 采用 Streamlit 构建移动端和 PC 端都可访问的轻量应用。
- 使用 ResNet18 迁移学习降低训练成本，提高小数据集识别效果。
- 引入 GPT-5 API，根据用户目标生成自然语言饮食建议。
- 支持历史记录和统计分析，便于展示完整业务闭环。
- 设计稿采用移动端轻食记录风，提供移动端 5 个独立页面和 PC 端 5 个宽屏页面，便于后续 Streamlit 页面实现。

## 8. UI 设计基准

本项目 UI 以 Open Design 最新稿和本地归档为准：

- Open Design 项目：`NutriSnap 轻食记录`
- Open Design 项目 ID：`0fbcf412-3ba4-4dbf-b769-d965b9ef7e69`

## 9. 系统边界

- 一期只识别 Food-101 子集中的 12 类食物。
- 系统不自动估计食物真实重量，重量由用户输入或选择默认值。
- GPT-5 只用于饮食建议文本生成，不参与食物图像识别。
- 建议内容不作为医学诊断、治疗方案或专业营养处方。
- 不做原生 Android 或 iOS App，移动端通过手机浏览器访问 Streamlit 应用。
