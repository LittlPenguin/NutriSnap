# 测试与验收方案

## 1. 测试目标

测试目标是确认系统可以稳定完成核心业务闭环：

```text
上传图片 → 食物识别 → 输入重量 → 热量估算 → GPT建议 → 历史记录 → 统计展示
```

同时确认系统在 PC 端和移动端浏览器中都具备可演示性。

## 2. 模型测试

| 测试项 | 方法 | 验收标准 |
|---|---|---|
| 模型加载 | 加载 `models/food_resnet18.pth` | 无报错 |
| 类别映射 | 读取 `models/class_names.json` | 12 个类别完整 |
| 测试集 Accuracy | 在 test 集运行评估 | 达到可演示水平 |
| Top-3 Accuracy | 判断正确类别是否在前三名 | 明显高于 Top-1 |
| 单图预测 | 使用演示图片测试 | 输出类别、置信度、Top-3 |

建议在报告中展示：

- 测试集准确率。
- 训练/验证曲线。
- 3 到 5 张预测示例。

## 3. 功能测试

| 编号 | 场景 | 输入 | 预期结果 |
|---|---|---|---|
| FT-01 | 上传 jpg 图片 | `.jpg` 食物图片 | 显示预览 |
| FT-02 | 上传 png 图片 | `.png` 食物图片 | 显示预览 |
| FT-03 | 上传非法文件 | `.txt` 文件 | 提示格式不支持 |
| FT-04 | 正常识别 | 支持类别图片 | 返回类别、置信度、Top-3 |
| FT-05 | 输入正常重量 | `150` | 返回估算热量 |
| FT-06 | 输入 0 或负数 | `0`、`-10` | 提示重量无效 |
| FT-07 | GPT 正常 | API Key 有效 | 返回 GPT 建议 |
| FT-08 | GPT 失败 | 移除 API Key | 返回本地规则建议 |
| FT-09 | 历史保存 | 完成一次识别 | 历史表新增记录 |
| FT-10 | 统计分析 | 多条历史记录 | 图表能显示汇总 |

## 4. 数据库测试

| 测试项 | 验收标准 |
|---|---|
| `food_calorie` 初始化 | 12 个支持类别都有热量数据 |
| 类别一致性 | 热量表 `class_name` 与模型类别映射一致 |
| 历史写入 | 完成识别后写入 `recognition_history` |
| GPT 日志 | 成功或失败均写入 `gpt_advice_log` |
| 日期汇总 | 能按 `created_at` 汇总每日热量 |

## 5. PC 端验收

测试浏览器：

- Microsoft Edge
- Google Chrome

验收标准：

- 页面可以正常打开。
- PC 端应覆盖 5 个宽屏页面：食物识别工作台、历史记录、食物热量表、统计分析、系统说明。
- 图片上传控件可用。
- 识别结果、重量输入、建议展示流程完整。
- 历史记录和统计图表能正常显示。
- 系统说明页不展示技术栈信息块。
- 页面主要内容不遮挡、不重叠。

## 6. 移动端验收

测试方式：

- 电脑和手机连接同一 Wi-Fi。
- 手机浏览器访问 `http://<电脑IP>:8501`。

验收标准：

- 手机可打开 Streamlit 应用。
- 移动端应覆盖 5 个独立页面：食物识别、历史记录、食物热量表、统计分析、系统说明。
- 可从相册或拍照上传图片。
- 页面采用单列阅读体验。
- 按钮和输入框可正常点击。
- 识别结果和建议文本不明显溢出。
- 历史记录表格即使横向滚动，也不影响核心流程。
- 系统说明页不展示技术栈卡片。

## 7. GPT 建议验收

| 测试项 | 验收标准 |
|---|---|
| 中文输出 | 建议为中文 |
| 长度控制 | 建议不超过 100 字 |
| 估算说明 | 明确热量为估算值 |
| 非医学诊断 | 不出现诊断、治疗、处方表达 |
| 失败降级 | API 异常时仍能返回本地建议 |

## 8. 文档与设计稿一致性验收

| 检查项 | 验收标准 |
|---|---|
| Open Design 本地归档 | `end/opendesign-nutrisnap/index.html` 与 Open Design 最新稿核心内容一致 |
| 移动端页面数量 | 设计稿包含移动端 5 个独立页面 |
| PC 端页面数量 | 设计稿包含 PC 端 5 个宽屏页面 |
| 系统说明页 | 移动端和 PC 端系统说明页均不展示技术栈 |
| 示例数据 | 统一使用披萨、150g、266 kcal / 100g、399 kcal |
| 旧示例清理 | 不再出现旧版错误热量示例 |
| 风险文案 | 明确热量为估算值，不作为医学或营养诊断 |

## 10. 验收通过标准

系统达到以下条件即可认为满足一期课程项目要求：

- 完成 Food-101 子集食物识别。
- 能根据用户输入重量计算估算热量。
- 能生成 GPT 或本地规则饮食建议。
- 能保存并展示历史记录。
- 能在 PC 和手机浏览器中完成演示。
- 实现页面与 Open Design 设计稿保持一致，尤其是移动端 5 页面、PC 端 5 页面和系统说明页不展示技术栈。
- 文档明确说明系统局限，不夸大热量估算准确性。
