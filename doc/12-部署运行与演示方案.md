# 部署运行与演示方案

## 1. 本地运行目标

本项目以本地运行为主，使用 Streamlit 启动 Web App。PC 端通过浏览器访问，移动端通过同一局域网中的手机浏览器访问。

系统不要求部署到云服务器，也不要求打包成原生 Android 或 iOS 应用。

## 2. 环境准备

建议开发环境：

| 项目 | 建议 |
|---|---|
| 操作系统 | Windows |
| Python 环境 | Miniconda |
| IDE | PyCharm |
| 浏览器 | Edge / Chrome |
| 移动端 | 手机浏览器 |

创建环境：

```text
conda create -n food-calorie python=3.10
conda activate food-calorie
pip install -r requirements.txt
```

## 3. 配置文件

`.env.example` 示例：

```text
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-5
OPENAI_TIMEOUT_SECONDS=20
```

实际开发时复制为 `.env`：

```text
.env
```

注意：

- `.env` 中不要写入文档或截图。
- 如果 GPT-5 模型权限不可用，应通过 `OPENAI_MODEL` 改为当前账号可用且教师允许的 GPT-5 系列模型。

## 4. 本地启动

启动命令：

```text
conda activate food-calorie
streamlit run app.py
```

PC 端访问：

```text
http://localhost:8501
```

## 5. 手机访问

前提：

- 电脑和手机连接同一 Wi-Fi。
- Windows 防火墙允许局域网访问 Streamlit 端口。

步骤：

1. 在电脑上查看局域网 IP。
2. 启动 Streamlit。
3. 手机浏览器访问：

```text
http://<电脑IP>:8501
```

示例：

```text
http://192.168.1.20:8501
```

## 6. 演示图片准备

建议准备固定演示图片：

```text
assets/demo_images/
  pizza_demo.jpg
  hamburger_demo.jpg
  sushi_demo.jpg
```

演示图片应满足：

- 类别属于系统支持的 12 类。
- 图片主体清晰。
- 背景不要过于复杂。
- 图片大小适中，避免上传过慢。

## 7. 5 分钟视频演示脚本

建议流程：

| 时间 | 内容 |
|---|---|
| 0:00-0:30 | 介绍项目背景和目标 |
| 0:30-1:00 | 口播介绍技术栈：Streamlit、PyTorch、ResNet18、SQLite、GPT-5 |
| 1:00-2:00 | 上传披萨或汉堡图片，展示识别结果和 Top-3 |
| 2:00-2:40 | 输入重量，展示估算热量 |
| 2:40-3:20 | 选择用户目标，生成 GPT 饮食建议 |
| 3:20-4:00 | 查看历史记录和热量统计 |
| 4:00-4:40 | 手机浏览器访问演示 |
| 4:40-5:00 | 总结系统特点和局限 |

演示 UI 时应与 Open Design 最新稿保持一致：

- 移动端展示移动端 5 个独立页面：食物识别、历史记录、食物热量表、统计分析、系统说明。
- PC 端展示 PC 端 5 个宽屏页面：食物识别工作台、历史记录、食物热量表、统计分析、系统说明。
- 系统说明页只展示项目简介、使用步骤、功能边界和免责声明，不展示技术栈卡片或技术栈信息块。
- 技术栈可以在视频口播、报告或答辩材料中介绍。
- 演示示例统一使用披萨、150g、266 kcal / 100g、399 kcal。

## 8. 答辩讲解重点

答辩时建议重点讲：

- 为什么选择食物图像识别和热量估算作为应用场景。
- ResNet18 迁移学习如何降低训练成本。
- 热量估算为什么需要用户输入重量。
- GPT-5 在系统中负责生成建议，而不是识别图片。
- 系统如何兼容 PC 和移动端浏览器。
- 为什么 UI 设计采用移动端 5 页面和 PC 端 5 页面。
- 为什么系统说明页不展示技术栈，而将技术栈放在报告和口播中介绍。
- 系统局限：类别有限、热量是估算值、不能替代专业营养建议。

## 9. 常见问题处理

| 问题 | 处理 |
|---|---|
| 手机打不开页面 | 检查是否同一 Wi-Fi、电脑 IP 是否正确、防火墙是否拦截 |
| GPT 调用失败 | 展示本地规则建议，说明系统支持降级 |
| 模型预测错误 | 展示 Top-3，说明模型受数据集和图片质量影响 |
| 图片上传慢 | 更换较小图片或压缩图片 |
| 热量不准确 | 说明热量来自参考表，结果为辅助估算 |
