# GPT 饮食建议功能设计

## 1. 功能定位

GPT 饮食建议是本项目的正式功能，但不是食物识别主模型。系统的 AI 分工如下：

| 能力 | 技术 | 职责 |
|---|---|---|
| 食物图像识别 | PyTorch + ResNet18 | 判断图片中的食物类别 |
| 热量估算 | SQLite 热量表 + 计算公式 | 根据类别和重量估算热量 |
| 饮食建议 | GPT-5 API | 生成自然语言建议 |

GPT-5 不参与模型训练，不替代 ResNet18，也不负责从图片中直接识别食物。

## 2. 使用场景

当用户完成食物识别和热量估算后，系统根据以下信息生成饮食建议：

- 食物中文名称。
- 用户输入重量。
- 每 100g 热量数据。
- 估算总热量。
- 用户饮食目标。
- 系统限制说明。

用户目标默认提供：

| 目标 | 说明 |
|---|---|
| 普通饮食 | 给出均衡饮食建议 |
| 减脂 | 关注热量控制和低脂搭配 |
| 增肌 | 关注蛋白质和训练后补充 |
| 控糖 | 避免高糖、高精制碳水建议，但不做医学判断 |

## 3. 输入数据

GPT 服务接收结构化输入：

```python
{
    "food_name": "披萨",
    "weight_g": 150,
    "calorie_per_100g": 266,
    "total_calorie": 399,
    "user_goal": "减脂"
}
```

不发送用户姓名、手机号、学号、图片文件、精确位置等隐私信息。

## 4. 输出数据

返回结构：

```python
{
    "status": "success",
    "advice": "这份披萨热量约 399 kcal，减脂期间建议控制份量，搭配蔬菜和无糖饮品，并减少同餐高脂食物。"
}
```

输出要求：

- 中文。
- 100 字以内。
- 明确热量为估算值。
- 不出现医学诊断。
- 不使用绝对化表述，例如“必须”“一定能治愈”。
- 给出 1 到 2 条可执行建议。

## 5. Prompt 模板

建议 Prompt：

```text
你是一个饮食记录应用中的健康饮食建议助手。

请根据以下信息生成中文饮食建议：
- 食物：{food_name}
- 用户输入重量：{weight_g}g
- 每 100g 热量：{calorie_per_100g} kcal
- 系统估算热量：{total_calorie} kcal
- 用户目标：{user_goal}

要求：
1. 100字以内。
2. 明确说明热量是估算值。
3. 给出可执行的饮食搭配或份量建议。
4. 不提供医学诊断、治疗建议或处方。
5. 如果涉及控糖，只做一般饮食提醒，不替代医生建议。
```

## 6. 配置项

建议使用 `.env` 保存配置：

```text
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-5
OPENAI_TIMEOUT_SECONDS=20
```

说明：

- `.env` 不应提交到公开源码或压缩包中。
- 提交项目时提供 `.env.example`，只写变量名，不写真实密钥。
- 如果当前账号没有指定 GPT-5 模型权限，后续实现时应通过 `OPENAI_MODEL` 修改为账号可用的 GPT-5 系列模型或教师允许的替代模型。

## 7. 调用流程

```text
食物识别成功
  ↓
热量估算成功
  ↓
用户选择目标
  ↓
构造 Prompt
  ↓
调用 GPT-5 API
  ↓
返回建议
  ↓
保存到 recognition_history 和 gpt_advice_log
```

## 8. 失败降级

GPT 调用失败时，系统必须继续可用。

| 失败原因 | 页面行为 | 日志状态 |
|---|---|---|
| 未配置 API Key | 显示本地规则建议 | `fallback` |
| 网络超时 | 显示本地规则建议 | `fallback` |
| API 返回错误 | 显示本地规则建议 | `error` |
| 输出为空 | 显示本地规则建议 | `fallback` |

本地规则建议示例：

| 热量范围 | 建议 |
|---|---|
| `< 200 kcal` | 本次估算热量较低，可作为轻食或加餐参考。 |
| `200-500 kcal` | 本次估算热量适中，建议搭配蔬菜和饮水，注意全天总摄入。 |
| `> 500 kcal` | 本次估算热量较高，建议控制份量，减少同餐高脂和高糖食物。 |

## 9. 安全与隐私

- 不上传图片原文件到 GPT。
- 不上传用户身份信息。
- 不保存完整敏感 Prompt，只保存 `prompt_summary`。
- 页面中说明建议仅供饮食参考，不构成医学或营养诊断。

## 10. 与设计稿同步要求

GPT 建议模块在界面中的展示应与 Open Design 最新稿一致：

- 示例食物统一为披萨。
- 示例重量统一为 `150g`。
- 示例热量数据统一为 `266 kcal / 100g` 和 `399 kcal`。
- GPT 建议文案示例为“这份披萨热量约 399 kcal，减脂期间建议控制份量，搭配蔬菜和无糖饮品。”
- GPT 失败时显示“本地规则建议”，系统仍保存可用记录。
- GPT-5 只负责饮食建议，不参与图像识别。
- 移动端和 PC 端系统说明页面不展示技术栈；GPT 功能边界可在系统说明页中展示。

## 11. 验收标准

- GPT 配置正确时，能生成符合要求的中文建议。
- GPT 不可用时，系统能自动显示本地规则建议。
- 建议内容不超过 100 字。
- 建议内容不包含诊断、治疗或处方性质表达。
- GPT 调用状态能写入 `gpt_advice_log`。
