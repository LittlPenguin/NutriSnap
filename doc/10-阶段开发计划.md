# 阶段开发计划

## 1. 开发目标

按照“先规则与可验证基础、再 UI 高保真对齐、最后验收收尾”的原则推进开发。每个阶段都必须产生可检查输出物、更新本文件完成状态，并使用中文 Git 提交。

最终交付物包括：

- Streamlit 应用源码。
- Food-101 子集整理脚本。
- ResNet18 模型训练与评估脚本。
- SQLite 数据库与初始化数据。
- GPT 饮食建议功能。
- Open Design 风格对齐的 PC / 移动端页面。
- 测试记录、阻塞记录、课程设计报告和演示素材。
