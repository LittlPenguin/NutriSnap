# 数据集与数据处理方案

## 1. 数据集选择

本项目使用 **Food-101 子集** 作为食物图像识别数据来源。Food-101 是常用食物图像分类数据集，包含多种食物类别，适合用于迁移学习和课程项目演示。

由于完整 Food-101 类别较多，训练和整理成本较高，本项目一期只选择 12 个类别作为支持范围，以保证训练、演示和文档撰写都可控。

## 2. 一期支持类别

| 英文类别 | 中文名称 | 食物类型 |
|---|---|---|
| `apple_pie` | 苹果派 | 甜点 |
| `baby_back_ribs` | 烤肋排 | 肉类 |
| `beef_carpaccio` | 牛肉薄片 | 肉类 |
| `caesar_salad` | 凯撒沙拉 | 沙拉 |
| `cheesecake` | 芝士蛋糕 | 甜点 |
| `chicken_curry` | 咖喱鸡 | 主菜 |
| `dumplings` | 饺子 | 主食 |
| `french_fries` | 薯条 | 小吃 |
| `hamburger` | 汉堡 | 快餐 |
| `pizza` | 披萨 | 快餐 |
| `ramen` | 拉面 | 主食 |
| `sushi` | 寿司 | 主食 |

## 3. 类别选择原则

- 食物外观具有一定差异，便于模型学习。
- 覆盖主食、快餐、甜点、肉类、沙拉等不同类别。
- 适合演示热量差异，例如沙拉、披萨、汉堡、芝士蛋糕。
- 支持后续扩展，不影响系统整体架构。

## 4. 数据目录结构

建议整理后的数据目录如下：

```text
dataset/
  food101_subset/
    train/
      apple_pie/
      baby_back_ribs/
      ...
    val/
      apple_pie/
      baby_back_ribs/
      ...
    test/
      apple_pie/
      baby_back_ribs/
      ...
```

每个类别目录下存放对应食物图片。

## 5. 数据划分

建议按照以下比例划分：

| 数据集 | 比例 | 用途 |
|---|---:|---|
| 训练集 | 70% | 模型参数训练 |
| 验证集 | 15% | 调参和选择最优模型 |
| 测试集 | 15% | 最终评估模型效果 |

如果单类图片数量有限，应保证每个类别在训练、验证、测试集中都有样本，避免某一类别只出现在训练集中。

## 6. 图像预处理

训练和推理阶段都需要统一输入格式。

### 6.1 训练阶段

建议使用以下处理：

```text
随机裁剪或中心裁剪
调整尺寸到 224×224
随机水平翻转
可选颜色扰动
转换为 Tensor
使用 ImageNet mean/std 归一化
```

### 6.2 推理阶段

推理阶段保持确定性：

```text
读取图片
转换为 RGB
调整尺寸到 224×224
转换为 Tensor
使用 ImageNet mean/std 归一化
```

推理阶段不做随机增强，避免同一张图片多次预测结果不一致。

## 7. 类别映射文件

建议创建 `models/class_names.json`，用于保存模型输出索引和类别名称之间的关系。

示例结构：

```json
[
  {"index": 0, "class_name": "apple_pie", "name_cn": "苹果派"},
  {"index": 1, "class_name": "baby_back_ribs", "name_cn": "烤肋排"},
  {"index": 2, "class_name": "beef_carpaccio", "name_cn": "牛肉薄片"}
]
```

该文件同时供训练脚本、预测服务和热量表校验使用。

## 8. 热量数据准备

热量数据不来自 Food-101，需要单独维护。建议建立 `data/food_calorie_seed.csv`，包含：

| 字段 | 说明 |
|---|---|
| `class_name` | Food-101 英文类别 |
| `name_cn` | 中文名称 |
| `calorie_per_100g` | 每 100g 估算热量 |
| `default_weight_g` | 默认份量 |
| `category` | 食物类型 |
| `note` | 备注 |

热量值用于课程项目估算，应在系统说明中标明数据为参考值。

## 9. 数据质量检查

整理数据集后需要检查：

- 每个类别都有训练、验证和测试图片。
- 类别目录名与 `class_names.json` 一致。
- 图片能被 Pillow 正常读取。
- 热量表中的 `class_name` 覆盖全部 12 个支持类别。
- 不同类别图片没有明显错放。

## 10. 后续扩展

如果一期功能稳定，可以扩展：

- 增加 Food-101 更多类别。
- 引入中文菜品数据集。
- 增加多标签或目标检测数据，支持一张图中多个食物。
- 增加用户自定义食物热量表。
