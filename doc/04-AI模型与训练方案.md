# AI 模型与训练方案

## 1. 模型任务定义

本项目的 AI 模型任务是 **食物图像分类**。模型输入一张食物图片，输出该图片最可能属于的 Food-101 子集类别，并给出预测置信度和 Top-3 候选类别。

模型不负责估计食物重量，也不直接输出热量。热量估算由业务模块根据食物类别、用户输入重量和热量数据库完成。

## 2. 模型选择

一期模型选择 **ResNet18**。

选择理由：

- ResNet18 是经典卷积神经网络，适合作为课程设计中的图像识别模型。
- 残差连接结构有利于稳定训练，比普通 CNN 更适合讲解技术原理。
- 可直接使用 ImageNet 预训练权重进行迁移学习，减少训练时间。
- 模型规模适中，适合在普通 PC 环境进行推理演示。
- PyTorch 和 torchvision 对 ResNet18 支持完善，后续开发成本低。

## 3. ResNet18 原理概述

ResNet 的核心思想是 **残差学习**。普通深层神经网络在层数增加后可能出现梯度消失、退化等问题。ResNet 通过残差连接将输入特征直接传递到后续层，使网络学习输入与输出之间的残差，从而提升训练稳定性。

简化表示：

```text
输出 = F(x) + x
```

其中：

- `x` 是输入特征。
- `F(x)` 是卷积层学习到的残差映射。
- `F(x) + x` 是残差块输出。

在本项目中，ResNet18 用于从食物图片中提取颜色、纹理、形状和局部结构特征，并通过分类层判断食物类别。

## 4. 迁移学习方案

本项目不从零训练模型，而是使用 ImageNet 预训练 ResNet18 做迁移学习。

训练流程：

1. 加载 torchvision 提供的 ResNet18 预训练权重。
2. 保留模型主体卷积层作为通用图像特征提取器。
3. 将最后的全连接层替换为 Food-101 子集类别数。
4. 第一阶段冻结主干网络，只训练新的分类层。
5. 第二阶段可选解冻后几层，使用较小学习率微调模型。
6. 使用验证集准确率选择最优模型。
7. 保存模型权重和类别映射。

## 5. 输入输出规格

### 5.1 输入规格

| 项目 | 要求 |
|---|---|
| 输入类型 | RGB 食物图片 |
| 支持格式 | jpg、jpeg、png |
| 输入尺寸 | 统一处理为 `224×224` |
| 归一化 | 使用 ImageNet 均值和标准差 |

归一化参数：

```text
mean = [0.485, 0.456, 0.406]
std  = [0.229, 0.224, 0.225]
```

### 5.2 输出规格

模型推理服务输出：

| 字段 | 类型 | 说明 |
|---|---|---|
| `predicted_class` | str | Food-101 英文类别名 |
| `predicted_name_cn` | str | 中文类别名 |
| `confidence` | float | 最高预测类别置信度 |
| `top3` | list | 前三名类别及置信度 |

示例：

```python
{
    "predicted_class": "pizza",
    "predicted_name_cn": "披萨",
    "confidence": 0.93,
    "top3": [
        {"class_name": "pizza", "name_cn": "披萨", "confidence": 0.93},
        {"class_name": "hamburger", "name_cn": "汉堡", "confidence": 0.04},
        {"class_name": "cheesecake", "name_cn": "芝士蛋糕", "confidence": 0.01}
    ]
}
```

## 6. 训练数据

模型训练使用 Food-101 子集，一期默认 12 类：

```text
apple_pie
baby_back_ribs
beef_carpaccio
caesar_salad
cheesecake
chicken_curry
dumplings
french_fries
hamburger
pizza
ramen
sushi
```

每个类别从 Food-101 原始数据中筛选图片，并划分为训练集、验证集和测试集。

## 7. 训练配置建议

| 配置项 | 建议值 |
|---|---|
| 模型 | ResNet18 |
| 损失函数 | CrossEntropyLoss |
| 优化器 | Adam 或 SGD |
| 初始学习率 | `1e-3` 训练分类头，`1e-4` 微调主干 |
| batch size | 16 或 32，视显存/内存调整 |
| epoch | 10 到 20 |
| 输入尺寸 | `224×224` |
| 保存策略 | 验证集准确率最高时保存 |

## 8. 评估指标

| 指标 | 说明 |
|---|---|
| Accuracy | 测试集中预测正确的比例 |
| Top-3 Accuracy | 正确类别出现在前三个候选中的比例 |
| Loss 曲线 | 观察训练和验证损失变化 |
| 混淆矩阵 | 可选，用于观察易混类别 |

课程项目中建议至少展示：

- 训练集/验证集准确率曲线。
- 测试集 Accuracy。
- 若干图片预测示例。

## 9. 模型文件

后续开发中模型相关文件放置在：

```text
models/
  food_resnet18.pth
  class_names.json
```

说明：

- `food_resnet18.pth` 保存训练好的模型权重。
- `class_names.json` 保存类别索引、英文类别名和中文类别名映射。

## 10. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| 类别相似导致误识别 | 影响识别准确率 | 显示 Top-3，答辩时说明模型局限 |
| 数据量不足 | 模型泛化较差 | 使用 Food-101 子集并做数据增强 |
| 训练耗时过长 | 影响开发进度 | 使用预训练权重和较少类别 |
| 模型文件缺失 | 应用无法识别图片 | 页面提示“模型未加载”，避免程序崩溃 |
