# 功能模块设计

## 1. 模块划分

系统按业务流程划分为七个功能模块：

| 模块 | 主要职责 |
|---|---|
| 图片上传模块 | 接收、校验并预览用户上传的食物图片 |
| 食物识别模块 | 调用 ResNet18 模型识别食物类别 |
| 热量估算模块 | 根据类别和重量计算估算热量 |
| GPT 饮食建议模块 | 根据结果生成个性化建议 |
| 历史记录模块 | 保存和查询识别历史 |
| 食物热量表模块 | 管理和展示食物热量数据 |
| 统计分析模块 | 汇总历史热量并生成图表 |

模块界面展示应与 Open Design 最新稿一致：移动端 5 个独立页面，PC 端 5 个宽屏页面。系统说明页只展示项目简介、使用步骤、功能边界和免责声明，不展示技术栈。

## 2. 图片上传模块

### 2.1 功能说明

图片上传模块负责接收用户在 PC 或手机浏览器中上传的食物图片，并在页面中显示预览图。

### 2.2 输入

- 图片文件：`jpg`、`jpeg`、`png`
- 上传来源：PC 文件选择、手机相册、手机拍照

### 2.3 输出

- 图片预览
- 可供模型推理使用的 `PIL.Image.Image` 对象
- 错误提示，例如文件格式不支持或图片无法读取

### 2.4 规则

- 限制文件类型为图片。
- 建议限制图片大小，例如不超过 10MB。
- 上传后不立即保存历史，只有完成识别和热量计算后才保存记录。

## 3. 食物识别模块

### 3.1 功能说明

食物识别模块加载训练好的 ResNet18 模型，对上传图片进行分类预测。

### 3.2 处理流程

```text
读取图片
  ↓
转换 RGB
  ↓
resize 到 224×224
  ↓
归一化
  ↓
模型推理
  ↓
Softmax 概率计算
  ↓
输出 Top-3
```

### 3.3 输出字段

| 字段 | 说明 |
|---|---|
| `predicted_class` | Food-101 英文类别 |
| `predicted_name_cn` | 中文类别名 |
| `confidence` | 最高类别置信度 |
| `top3` | 前三名预测类别和置信度 |

## 4. 热量估算模块

### 4.1 功能说明

热量估算模块根据识别出的食物类别查询本地热量数据库，并结合用户输入重量计算总热量。

### 4.2 公式

```text
估算热量 = 每100g热量 × 用户输入重量 / 100
```

### 4.3 示例

```text
食物：披萨
每100g热量：266 kcal / 100g
输入重量：150g
估算热量：266 × 150 / 100 = 399 kcal
```

### 4.4 边界处理

- 重量为空时使用默认重量。
- 重量小于等于 0 时提示用户重新输入。
- 食物类别未在热量表中找到时提示“暂无热量数据”。

## 5. GPT 饮食建议模块

### 5.1 功能说明

该模块使用 GPT-5 API，根据食物名称、重量、热量和用户目标生成简短饮食建议。

### 5.2 用户目标

- 普通饮食
- 减脂
- 增肌
- 控糖

### 5.3 输出要求

- 使用中文。
- 控制在 100 字以内。
- 明确热量为估算值。
- 不提供医学诊断。
- 给出一条可执行建议。

### 5.4 降级逻辑

当 API Key 未配置、网络错误、超时或接口异常时，系统返回本地规则建议。例如：

- 总热量小于 200 kcal：热量较低，可作为轻食或加餐参考。
- 总热量 200 到 500 kcal：热量适中，注意整体饮食搭配。
- 总热量超过 500 kcal：热量较高，建议控制同餐油脂和主食摄入。

## 6. 历史记录模块

### 6.1 保存内容

- 图片文件名
- 预测类别
- 中文名称
- 置信度
- 用户输入重量
- 每 100g 热量
- 总热量
- GPT 建议
- 创建时间

### 6.2 展示方式

- 按时间倒序展示。
- PC 端使用表格展示完整历史记录。
- 移动端使用紧凑卡片展示关键字段，避免过宽表格影响阅读。
- 可按日期汇总热量。

## 7. 食物热量表模块

### 7.1 功能说明

展示系统支持的 Food-101 子集类别及对应热量数据。

### 7.2 字段

- 英文类别名
- 中文名称
- 食物类别
- 每 100g 热量
- 默认重量
- 备注

### 7.3 使用方式

该表既用于系统计算，也用于用户查看数据来源和系统支持范围。

## 8. 统计分析模块

### 8.1 功能说明

统计分析模块基于历史记录展示摄入热量概况。

### 8.2 图表

- 历史总热量。
- 今日识别次数。
- 按日期汇总的热量柱状图。
- 食物类别出现次数排行。

## 9. 功能优先级

| 优先级 | 功能 |
|---|---|
| P0 | 上传图片、识别类别、输入重量、计算热量 |
| P1 | GPT 建议、历史记录、热量表 |
| P2 | 统计图表、系统说明、移动端细节优化 |

## 10. 与设计稿同步要求

- 食物识别示例统一使用披萨、150g、266 kcal / 100g、399 kcal。
- 识别结果展示 Top-3：披萨 93%、汉堡 4%、芝士蛋糕 1%。
- 移动端包含食物识别、历史记录、食物热量表、统计分析、系统说明 5 个页面。
- PC 端包含食物识别工作台、历史记录、食物热量表、统计分析、系统说明 5 个页面。
- 系统说明页不展示技术栈；技术栈只在项目文档、报告和演示讲解中介绍。
