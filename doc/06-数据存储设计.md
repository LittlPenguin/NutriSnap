# 数据存储设计

## 1. 存储方案

本项目使用 SQLite 作为本地数据存储方案。SQLite 不需要单独部署数据库服务，适合课程项目、单机演示和轻量应用。

数据库主要保存：

- 食物热量基础数据。
- 用户识别历史。
- GPT 饮食建议日志。

模型权重和类别映射以本地文件形式保存，不写入 SQLite。

## 2. 数据库文件

建议数据库文件路径：

```text
data/food_calorie.db
```

初始化数据来源：

```text
data/food_calorie_seed.csv
models/class_names.json
```

## 3. ER 关系说明

```text
food_calorie
  1 ──── n
recognition_history
  1 ──── n
gpt_advice_log
```

说明：

- 一种食物类别可以对应多条识别历史。
- 一条识别历史可以对应一条或多条 GPT 建议日志。
- `recognition_history.predicted_class` 与 `food_calorie.class_name` 逻辑关联。
- `gpt_advice_log.history_id` 关联 `recognition_history.id`。

## 4. 表：food_calorie

该表保存食物热量基础数据。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 食物编号 |
| `class_name` | TEXT | UNIQUE NOT NULL | Food-101 英文类别名 |
| `name_cn` | TEXT | NOT NULL | 中文名称 |
| `calorie_per_100g` | REAL | NOT NULL | 每 100g 热量 |
| `default_weight_g` | REAL | NOT NULL | 默认重量 |
| `category` | TEXT | NOT NULL | 食物类别 |
| `note` | TEXT | NULL | 备注 |

示例数据：

| class_name | name_cn | calorie_per_100g | default_weight_g | category |
|---|---|---:|---:|---|
| pizza | 披萨 | 266 | 150 | 快餐 |
| hamburger | 汉堡 | 295 | 180 | 快餐 |
| caesar_salad | 凯撒沙拉 | 180 | 120 | 沙拉 |

## 5. 表：recognition_history

该表保存每次识别和热量估算结果。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 历史编号 |
| `image_name` | TEXT | NOT NULL | 图片文件名 |
| `predicted_class` | TEXT | NOT NULL | 预测类别英文名 |
| `predicted_name_cn` | TEXT | NOT NULL | 中文名称 |
| `confidence` | REAL | NOT NULL | 置信度 |
| `weight_g` | REAL | NOT NULL | 用户输入重量 |
| `calorie_per_100g` | REAL | NOT NULL | 每 100g 热量 |
| `total_calorie` | REAL | NOT NULL | 总热量 |
| `gpt_advice` | TEXT | NULL | GPT 或本地建议 |
| `created_at` | TEXT | NOT NULL | 创建时间 |

## 6. 表：gpt_advice_log

该表保存 GPT 建议调用结果和降级状态。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 建议编号 |
| `history_id` | INTEGER | NULL | 对应历史记录 ID |
| `user_goal` | TEXT | NOT NULL | 用户目标 |
| `prompt_summary` | TEXT | NOT NULL | Prompt 摘要，不保存敏感信息 |
| `advice` | TEXT | NOT NULL | 返回建议 |
| `status` | TEXT | NOT NULL | `success` / `fallback` / `error` |
| `created_at` | TEXT | NOT NULL | 创建时间 |

## 7. 数据写入时机

| 场景 | 写入表 |
|---|---|
| 初始化系统 | `food_calorie` |
| 完成识别和热量计算 | `recognition_history` |
| GPT 成功返回建议 | `gpt_advice_log`，并更新历史建议 |
| GPT 失败使用本地建议 | `gpt_advice_log` 记录 `fallback` |

## 8. 数据校验规则

- `class_name` 必须存在于 `class_names.json`。
- `calorie_per_100g` 必须大于 0。
- `default_weight_g` 必须大于 0。
- `weight_g` 必须大于 0。
- `confidence` 范围为 0 到 1。
- `status` 只能是 `success`、`fallback` 或 `error`。

## 9. 备份与重置

课程演示前建议准备：

- 一份干净的 `food_calorie.db`。
- 一份含少量演示历史记录的数据库。
- 一组固定演示图片。

如果演示过程中数据混乱，可删除历史记录表数据，但不删除 `food_calorie` 基础数据。
