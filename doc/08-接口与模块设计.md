# 接口与模块设计

## 1. 设计说明

本项目使用 Streamlit 构建单体 Web App，不需要单独设计 HTTP API。这里的“接口”指 Python 内部模块之间的函数接口。后续开发应按模块职责拆分代码，避免把所有逻辑堆在 `app.py` 中。

## 2. 推荐代码结构

```text
app.py
services/
  predictor.py
  calorie_service.py
  database.py
  gpt_advice_service.py
  image_utils.py
train/
  train_model.py
  prepare_food101_subset.py
  evaluate_model.py
models/
  food_resnet18.pth
  class_names.json
data/
  food_calorie.db
  food_calorie_seed.csv
```

## 3. 模块职责

| 模块 | 职责 |
|---|---|
| `app.py` | Streamlit 页面入口和流程编排 |
| `services/predictor.py` | 模型加载、图片预处理、食物预测 |
| `services/calorie_service.py` | 热量查询、热量计算、本地规则建议 |
| `services/database.py` | SQLite 初始化、查询、写入 |
| `services/gpt_advice_service.py` | GPT-5 Prompt 构造、调用和降级 |
| `services/image_utils.py` | 图片读取、格式校验、文件名处理 |
| `train/train_model.py` | 模型训练 |
| `train/prepare_food101_subset.py` | Food-101 子集整理 |
| `train/evaluate_model.py` | 模型评估 |

## 4. 预测接口

### 4.1 函数

```python
predict_image(image: PIL.Image.Image) -> dict
```

### 4.2 输入

| 参数 | 类型 | 说明 |
|---|---|---|
| `image` | `PIL.Image.Image` | 用户上传并成功读取的图片 |

### 4.3 输出

```python
{
    "predicted_class": "pizza",
    "predicted_name_cn": "披萨",
    "confidence": 0.93,
    "top3": [
        {"class_name": "pizza", "name_cn": "披萨", "confidence": 0.93},
        {"class_name": "hamburger", "name_cn": "汉堡", "confidence": 0.04},
        {"class_name": "cheesecake", "name_cn": "芝士蛋糕", "confidence": 0.01}
    ]
}
```

### 4.4 异常

- 模型文件不存在：返回明确错误，由页面显示“模型未加载”。
- 类别映射文件不存在：返回明确错误。
- 图片无法处理：提示用户更换图片。

## 5. 热量计算接口

### 5.1 函数

```python
calculate_calorie(class_name: str, weight_g: float) -> dict
```

### 5.2 输入

| 参数 | 类型 | 说明 |
|---|---|---|
| `class_name` | str | Food-101 英文类别名 |
| `weight_g` | float | 用户输入重量，单位 g |

### 5.3 输出

```python
{
    "class_name": "pizza",
    "name_cn": "披萨",
    "weight_g": 150,
    "calorie_per_100g": 266,
    "total_calorie": 399
}
```

### 5.4 规则

- `weight_g` 必须大于 0。
- `total_calorie` 保留 1 位小数或取整，两种方式在项目中保持一致。
- 如果找不到热量数据，返回错误状态，页面不继续生成建议。

## 6. GPT 建议接口

### 6.1 函数

```python
generate_gpt_advice(
    food_name: str,
    weight_g: float,
    total_calorie: float,
    user_goal: str
) -> dict
```

### 6.2 输入

| 参数 | 类型 | 说明 |
|---|---|---|
| `food_name` | str | 食物中文名 |
| `weight_g` | float | 食物重量 |
| `total_calorie` | float | 估算热量 |
| `user_goal` | str | 普通饮食、减脂、增肌、控糖 |

### 6.3 输出

```python
{
    "status": "success",
    "advice": "这份披萨热量较高，建议搭配蔬菜并减少同餐高脂食物摄入。"
}
```

失败降级输出：

```python
{
    "status": "fallback",
    "advice": "GPT 建议暂不可用。该食物热量较高，建议控制份量并搭配蔬菜。"
}
```

## 7. 历史保存接口

### 7.1 函数

```python
save_history(record: dict) -> int
```

### 7.2 输入字段

| 字段 | 说明 |
|---|---|
| `image_name` | 图片文件名 |
| `predicted_class` | 预测类别 |
| `predicted_name_cn` | 中文名 |

| `calorie_per_100g` | 每 100g 热量 |
| `total_calorie` | 总热量 |
| `gpt_advice` | 建议 |

### 7.3 输出

返回新插入的历史记录 ID。

## 8. 查询接口

| 函数 | 返回 |
|---|---|
| `get_food_calorie(class_name: str) -> dict` | 指定食物热量数据 |
| `list_food_calories() -> list[dict]` | 所有支持类别 |
| `list_history(limit: int = 100) -> list[dict]` | 最近历史记录 |
| `get_daily_calorie_summary() -> pandas.DataFrame` | 按日期汇总热量 |

## 9. 错误处理策略

| 错误 | 处理方式 |
|---|---|
| 图片为空 | 页面提示上传图片 |
| 图片格式错误 | 页面提示仅支持 jpg、jpeg、png |
| 模型加载失败 | 页面提示先训练或放置模型文件 |
| 类别映射缺失 | 页面提示类别配置错误 |
| 热量数据缺失 | 页面提示当前类别暂无热量数据 |
| GPT API 失败 | 使用本地规则建议并记录日志 |
| 数据库写入失败 | 页面提示保存失败，但仍展示当前结果 |
